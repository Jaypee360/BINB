b) Software Project Description  
Software Name: BINB  
BINB (Brandon Is Not Boring) is a web -based application designed to help people moving into 
Brandon (for example, immigrants and international students), as well as current residents, 
easily discover events and activities happening around the city. We, the  creators of this web -
based application, are international students, so we understand how it feels to arrive in a new 
city without knowing what it offers. This can become lonely and isolating, and is not a good way 
to begin life in a new place. Even for current residents, BINB encourages people to step outside 
their comfort zone and try something new, such as attending a party instead of only networking 
events.  
 Through BINB, we are not only offering opportunities to explore the city, but also bridging the 
gap between new residents and long -time residents of Brandon. The system will collect and 
display events from multiple public websites using web scraping. Even ts will be organized into 
categories such as parties, sports, business, and community activities. Users will also be able to 
subscribe to a weekly email newsletter summarizing events happening in Brandon each week.  
 
Main Features include: Event collection from multiple websites, Event categorization, Weekly 
email subscription, and a simple and friendly user interface  
Core Functionalities: Scraping events and activities from the web, displaying events in a user -
friendly format, allowing users to subscribe, and providing weekly updates on events going on in 
Brandon.  
 
Constraints: Limited development time (until the end of April), Web scraping limited to publicly 
available websites, Event accuracy depends on the external websites.  
 
Quality Requirements:  
Reliability – Event date, time, and location must be accurate.  
Performance – The website should not use excessive system resources.  
Maintainability – The system should be easy to update and modify after launch.  
Security – Safe handling and storage of user email addresses  
c) User Stories (Story Cards)  
Story Card 1: Viewing Events  
When the user opens BINB, the website displays a list of upcoming events happening in 
Brandon in order of date and time. Each event shows its date, time, category, and location. The 
user can scroll through the list of events. If there are no events availab le, the website displays a 
message stating that there are currently no events.  
 
Story Card 2: Filtering Events by Category  
When the user opens BINB, the website displays a list of upcoming events. If the user wants to 
attend a sports event, they click on the “Sports” category, and only sports events are displayed. 
If there are no events in that category, the website displays a  message saying there are 
currently no events in that category and suggests subscribing to the newsletter.  
 
Story Card 3: Email Subscription  
The user enters their email address into the subscription form. The system validates that the 
email is in the correct format. If valid, the system stores the email address. At the start of each 
week, the system generates a list of upcoming events and sends  it to subscribers. If the user 
chooses to unsubscribe, the system removes their email from the mailing list.  
 
Story Card 4: Suggested Events  
When the user opens BINB, the website displays a list of upcoming events. If the user 
frequently uses the site, the system shows suggested events based on previously searched 
categories before displaying the full list of events.  
d) Prioritization  
Due to the limited development timeframe (approximately 2.5 months) and the use of two 5 -
week releases, only the most essential features will be implemented first.  
Release 1 (First 5 weeks)  
Story Card 1 – Viewing Events: This is the core purpose of BINB. If users cannot view events, 
the application has no value. This feature must be implemented first.  
 
Story Card 2 – Filtering Events by Category: Users need an easy way to find events they are 
interested in, such as sports or business events. Categorization improves usability.  
 
Story Card 3 – Email Subscription: This feature increases user engagement and retention by 
keeping users informed even when they do not visit the website regularly.  
 
Release 2 (Second 5 weeks)  
Story Card 4 – Suggested Events: This feature is lower priority and will be implemented after 
the core system is functional.
